## 1단계 — 사이트 구축 (아티팩트 → React)

### 1-1. 아티팩트 분해 & 자산 추출
- 원본 번들에서 뷰 템플릿(112KB) + 컨트롤러 JSX(26KB)를 분리 추출.
- 임베드 폰트(JetBrains Mono / Space Grotesk / Pretendard, 약 22MB) → CDN(Google Fonts · Pretendard)으로 대체해 경량화.
- 원본은 `reference/rest06.original.html` 로 보존(요청대로 전체 업로드).

### 1-2. DC 템플릿 엔진 재구현 (`src/dcRuntime.jsx`)
원본 dc-runtime 템플릿 문법을 자체 React 렌더러로 1:1 포팅:
- `<sc-for list as>` 반복 · `<sc-if value>` 조건 · `{{ expr }}` 보간 · `$index` 루프 인덱스
- `onclick="{{ }}"` → onClick, `style-hover` → 마우스 호버, `style` → React style 객체
- 템플릿 1회 파싱 후 캐시, 표현식은 `with(scope)` 평가 + 캐싱.

### 1-3. 앱 컨트롤러 (`src/App.jsx`)
- 아티팩트의 `Component`(DCLogic) → `React.Component` 포팅: 테마 6색 · 다크/라이트 ·
  화면 라우팅(home/online/offline/cert/ai/community/corporate/about/detail/enroll/dashboard) · 카운터·로테이터 fx.
- React 리렌더가 덮어쓰는 테마/모드 CSS 변수는 `componentDidUpdate` 에서 항상 복원.

### 1-4. 인증 · 결제 (`src/lib/`)
- **인증** `auth.js`: Supabase OAuth 구글·카카오 로그인, 세션 구독, 프로필 upsert.
  네비 "로그인" 버튼 → 로그인/계정 오버레이 모달(React, 디자인 외부 레이어로 얹음).
- **결제** `payment.js`: 아임포트(PortOne v2) SDK 지연로드 + 결제요청, 결제·수강 기록.
  키 미설정 시 테스트 모드로 흐름 검증. 무료체험은 결제 없이 수강신청.
- 디자인 보존을 위한 최소 템플릿 패치: "수강 신청하기"→결제화면, "결제하고 시작하기"→`doPay`,
  신청자 정보→로그인 유저 정보.

### 1-5. 인프라
- `vite.config.js` base `/`, `public/CNAME` = rest06.dreamitbiz.com.
- `.github/workflows/deploy.yml`: main push → 빌드(시크릿 주입) → Pages 배포.
- `sql/schema.sql`: `rest06_profiles / rest06_payments / rest06_enrollments` + RLS.
- **GitHub Pages build_type 를 legacy → workflow 로 전환**(신규 Pages가 legacy로 시작되어 Actions 배포가 안 먹던 이슈 해결), CNAME 등록.

## 2단계 — 마이페이지 실데이터 바인딩

마이페이지(대시보드)의 하드코딩된 더미값을 Supabase 실데이터로 교체.

### 2-1. 데이터 레이어 (`src/lib/dashboard.js`)
- `fetchDashboard(user, courses)`: `rest06_enrollments`(수강내역)를 강의 카탈로그와 결합 +
  `rest06_profiles`(스트릭) 조회 → `{ list, stats }` 반환.
- `touchStreak(userId)`: 로그인 시 연속 학습일 계산(오늘=유지 / 어제=+1 / 그 이전=리셋).

### 2-2. 바인딩 대상
- **이름 · 아바타 이니셜** → 로그인 유저
- **통계 4종**:
  - 수강 중 = active/trial 수강 수
  - 완료 강의 = Σ(강의 레슨수 × 진도율) 완료 레슨 합계
  - 연속 학습 = 프로필 스트릭(로그인 시 갱신)
  - 수료증 = 진도율 100% 과정 수
- **수강 목록** → 내 수강 강의(진도율 바), 클릭 시 상세로 이동.
- 비로그인/Supabase 미연결 시 데모값으로 폴백 → 항상 정상 렌더.

### 2-3. 로딩 트리거
마이페이지 진입 · 로그인/세션 복원 · **결제 완료 후 자동 이동** 시 재조회
→ 방금 결제한 강의가 즉시 수강 목록에 반영.

### 2-4. 스키마 확장
- `rest06_enrollments.progress`(진도율 0~100)
- `rest06_profiles.streak_days` / `last_active`
- 모두 `alter table ... add column if not exists` 라 기존 테이블에도 안전.
